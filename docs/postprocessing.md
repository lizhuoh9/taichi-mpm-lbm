# Post-Processing And Visualization

Step 10 adds lightweight tools for reading simulation output snapshots and
validation summaries. These tools are intended for diagnostics, regression
review, and quick plots. They do not change solver behavior.

## Snapshot Formats

The output writer supports two analysis formats:

- NPZ: compressed NumPy data for Python post-processing.
- VTK: ParaView-compatible files for quick field and particle inspection.

NPZ is the primary format for tests and scripted summaries. VTK is a smoke
visualization format for external tools.

## Reading NPZ Snapshots

Simulation snapshots are written as `snapshot_XXXXXX.npz` files under an output
directory. List and inspect them with:

```python
from fsi.postprocess import list_npz_snapshots, snapshot_info

for path in list_npz_snapshots("outputs/coupled_output_snapshot"):
    print(snapshot_info(path).to_dict())
```

Each NPZ snapshot always includes metadata:

- `step`
- `time`
- `lbm_shape`
- `mpm_particle_count`

It may also include LBM fields, MPM particle arrays, and coupling diagnostics
depending on the active `OutputConfig` flags.

## Exporting Time Series

Use `extract_snapshot_timeseries()` to turn a directory of snapshots into scalar
diagnostic arrays:

```python
from fsi.postprocess import (
    extract_snapshot_timeseries,
    write_timeseries_csv,
    write_timeseries_json,
)

timeseries = extract_snapshot_timeseries("outputs/postprocess_snapshots")
write_timeseries_csv(timeseries, "outputs/postprocess_snapshots/timeseries.csv")
write_timeseries_json(timeseries, "outputs/postprocess_snapshots/timeseries.json")
```

The extractor includes keys only when the source arrays exist. This means it can
read snapshots written with LBM fields, MPM particles, or coupling fields
disabled.

## Plotting Diagnostics

Basic PNG line plots are available through a lazy matplotlib import:

```python
from fsi.postprocess import extract_snapshot_timeseries, plot_timeseries

timeseries = extract_snapshot_timeseries("outputs/postprocess_snapshots")
plot_timeseries(
    timeseries,
    "mpm_center_of_mass_x",
    "outputs/postprocess_snapshots/mpm_com_x.png",
)
```

Matplotlib is a development dependency. It is not imported during normal
`import fsi`.

## Coupling Diagnostics

Step 9 coupling diagnostics can be summarized when snapshots were written with
`write_coupling_fields=True`:

```python
from fsi.postprocess import summarize_coupling_diagnostics

diagnostics = summarize_coupling_diagnostics("outputs/postprocess_snapshots")
print(diagnostics.keys())
```

Available coupling series may include:

- `coupling_force_norm`
- `total_particle_coupling_force_norm`
- `total_fluid_coupling_force_norm`
- `coupling_unsupported_particle_count`
- `coupling_partial_support_particle_count`
- `coupling_clipped_particle_count`
- `ib_force_norm`
- `ib_total_force_norm`
- `ib_active_cell_count`
- `ib_clipped_cell_count`
- `contact_candidate_count`
- `contact_damped_particle_count`

The Step 12 immersed-boundary/contact MVP writes optional IB/contact fields when
`write_coupling_fields=True`. These fields are diagnostic regression outputs;
they do not imply a full no-slip immersed-boundary or rigid-body contact model.

## Validation Summaries

The validation benchmark example writes
`outputs/validation_benchmark_suite/validation_summary.json`. Flatten it to a
metric table with:

```python
from fsi.postprocess import (
    load_validation_summary,
    validation_summary_table,
    write_validation_summary_csv,
)

reports = load_validation_summary(
    "outputs/validation_benchmark_suite/validation_summary.json"
)
rows = validation_summary_table(reports)
write_validation_summary_csv(
    rows,
    "outputs/validation_benchmark_suite/validation_summary.csv",
)
```

Each output row contains the case name, case pass state, metric name, value,
bounds, metric pass state, units, and description.

## ParaView VTK Workflow

For VTK output, open files generated by `OutputConfig(output_format="vtk")` or
`output_format="both"`:

- `fluid_XXXXXX.vti` contains fluid cell data.
- `particles_XXXXXX.vtu` contains MPM particle point data.

Typical ParaView inspection steps:

1. Open `fluid_XXXXXX.vti`.
2. Color by `rho`, `velocity`, `force`, `coupling_force`,
   `solid_volume_fraction`, or `immersed_boundary_force` when available.
3. Open `particles_XXXXXX.vtu`.
4. Use glyphs for particle directions if desired.
5. Compare the same step number across fluid and particle files.

The repository does not automate ParaView. Tests only verify that VTK files can
be written.

## Examples

Run:

```bash
python examples/postprocess_snapshots.py
python examples/postprocess_validation_summary.py
```

The snapshot example generates a tiny run, writes NPZ snapshots, exports
CSV/JSON time series, and writes a PNG plot under
`outputs/postprocess_snapshots/`.

The validation example runs the validation suite, writes
`validation_summary.json`, and exports a flattened `validation_summary.csv`
under `outputs/validation_benchmark_suite/`.

## Limitations

- Post-processing is diagnostic, not high-fidelity validation.
- VTK support is file output only, not ParaView automation.
- No interactive rendering or video generation is included.
- PNG plotting is intentionally basic and intended for quick trend inspection.
- Missing optional snapshot field groups are skipped rather than synthesized.
